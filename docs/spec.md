# 機能仕様書

## 機能概要
本システムは、MCP サーバーに関する情報を整理・蓄積し、検索・比較・参照できるナレッジベースを提供する。
利用者は各 MCP サーバーの用途、提供ツール、導入手順、認証方式、対応クライアント、参考リンク、更新履歴を確認できる。

公開配布版の主な利用者は、MCP サーバーを評価・運用したい開発者およびメンテナーとする。

MVP では以下を提供する。
- MCP サーバー情報の登録、更新、参照
- タグによる分類
- キーワード検索またはタグベースのカテゴリ絞り込み
- 検索サジェスト、ファセット、関連エントリ取得
- 詳細表示による導入判断支援
- 更新履歴の保持

本プロジェクトは GitHub で公開する OSS として開発し、ライセンスは MIT License とする。

## 利用シナリオ
1. 利用者はキーワードとタグベースのカテゴリを使って目的に合う MCP サーバーを検索する。
2. 利用者は検索中に候補サジェストとファセットを参照し、絞り込み条件を調整する。
3. 利用者は詳細表示で、対象サーバーの概要、提供ツール、セットアップ手順、認証有無、参考リンクを確認する。
4. 利用者は関連エントリ一覧から近い用途の MCP サーバーをたどる。
5. メンテナーは新しい MCP サーバー情報を登録する。
6. メンテナーは既存の MCP サーバー情報を更新し、タグを変更する。
7. システムは更新ごとに履歴を保持し、後から変更の事実を追跡できるようにする。

## 入力・出力の定義
### MCP サーバー情報
| 項目 | 必須 | 制約 | 説明 |
|:--|:--|:--|:--|
| 名前 | 必須 | 1 件のサーバーを識別できる名称であること | MCP サーバー名 |
| 概要 | 必須 | テキスト | サーバーの用途や特徴の要約 |
| タグ | 必須 | 1 件以上 5 件以下、後から変更可能 | 用途や特徴による分類 |
| 提供ツール一覧 | 必須 | 0 件以上の一覧 | サーバーが提供するツール名または要約 |
| セットアップ手順 | 必須 | テキストまたは段階的手順 | 導入手順、設定例、注意点 |
| 認証の有無 | 必須 | 真偽値または種別 | 認証不要、API キー、OAuth など |
| 対応クライアント | 任意 | 0 件以上の一覧 | 接続確認済みまたは想定クライアント |
| 参考リンク | 必須 | 1 件以上を推奨 | 仕様、README、配布元など |
| 更新履歴 | システム管理 | 1 件以上保持可能 | 更新日時、更新者、変更要約など |

### 検索入力
| 項目 | 必須 | 制約 | 説明 |
|:--|:--|:--|:--|
| キーワード | 任意 | テキスト | 名前、概要、ツール名などに対する検索語 |
| タグ | 任意 | 複数指定可 | タグ一致による絞り込み |
| カテゴリ | 任意 | MVP ではタグで表現する | 用途や分類による絞り込み |
| 認証条件 | 任意 | 実装方針に依存 | 認証不要、API キー、OAuth など |
| 対応クライアント | 任意 | 複数指定可 | 対応クライアントでの絞り込み |
| ツール名 | 任意 | 複数指定可 | 提供ツール名での絞り込み |
| 更新日時下限 | 任意 | UTC 日時 | 最近更新されたエントリへの絞り込み |
| 並び順 | 任意 | `relevance`, `updated-desc`, `name-asc` | 一覧の並び順 |

### 出力
- 検索結果一覧
	- 名前
	- 概要
	- タグ
	- 認証の有無
	- 主要ツールの要約
- 検索サジェスト
	- 候補文字列
	- 候補種別
	- スコア
- ファセット
	- タグ候補と件数
	- 認証方式候補と件数
	- 対応クライアント候補と件数
- 関連エントリ一覧
	- 関連エントリ識別子
	- 表示名
	- 関連度スコア
	- 関連理由
- 詳細表示
	- MCP サーバー情報の全項目
	- 更新履歴

## 制約・スコープ外
### 制約
- アーキテクチャは Medallion 構造と Ports and Adapters を組み合わせた疎結合構成とする。
- ドメインルールは特定の DB、Embedding、LLM SDK に依存してはならない。
- DB、Embedding、LLM はインターフェース越しに利用し、実装を差し替え可能にする。
- アプリケーションフレームワークは Microsoft Agent Framework を採用する。
- 永続化は EF Core を採用し、DB provider は差し替え可能にする。
- デフォルトの開発用 DB は SQLite とする。
- Embedding はオプション機能とし、MVP の必須要件には含めない。
- Embedding が無効でも検索、参照、更新、履歴機能は成立しなければならない。
- LLM が無効でも Bronze から Silver / Gold への整理は規則ベースで成立しなければならない。
- MVP ではカテゴリを独立マスタとして持たず、タグをカテゴリとして扱う。
- データモデルはサーバー単位を主とし、ツール情報はサーバー配下の子要素として扱う。
- タグは MVP では自由入力とし、候補管理は将来拡張とする。
- 更新履歴は日時、更新者、アクション、変更要約を保持し、差分本体は MVP の必須要件に含めない。
- グラフ構造は必要だが、MVP では専用グラフ DB を前提にしない。
- グラフ探索や経路計算が必要な場合は、永続化モデルとは分離してアプリケーション層で扱う。
- 検索 API は単純検索、複合条件検索、サジェスト、ファセット、関連エントリ取得を分離した構成とする。
- 公開配布版の seed data と curated ingest 対象は、再配布条件を確認できる公開情報に限定し、社内限定サーバーや非公開資料を含めない。
- 公開リポジトリには認証情報、社内限定情報、ライセンス不明な成果物を含めない。
- 公開ライセンスは MIT License とする。

### スコープ外
- 初期段階での自動クローリングや大規模収集基盤の構築
- MCP サーバーそのもののホスティングや運用監視
- ユーザーごとの複雑な認可や権限管理
- SNS 的なコメント、評価、レビュー機能
- 初期段階での多言語対応
- 初期段階での高度な AI 要約、自動分類、自動推薦
- 初期段階での RDF 基盤や専用グラフ DB の導入

## 将来検討事項
- 情報更新は手入力中心か、GitHub などからの半自動取り込みを行うか。
- 関連エントリの算出ロジックを共有タグ中心にするか、GraphProjection をどこまで使うか。
- グラフ構造は関連サーバー推薦、依存関係表現、用途間関係のどこまでを対象にするか。
- MVP のインターフェースは Web UI、CLI、API のどれを起点にするか。

## 非機能要件
### 保守性
- DB、Embedding、LLM の実装差し替え時に、ドメイン層と API 契約を変更しなくてよいこと。
- Bronze / Silver / Gold の各層は責務を明確に分離し、更新時の影響範囲を局所化すること。

### 拡張性
- 将来的に PostgreSQL、Azure SQL などの DB へ移行できること。
- 将来的に `text-embedding-3-small`, `text-embedding-3-large`, OSS Embedding 実装などへ切り替えできること。
- 将来的に Foundry、Azure OpenAI、OpenAI 互換プロバイダーへ LLM 実装を切り替えできること。

### 可観測性
- 整理処理の実行結果は履歴とログで追跡できること。
- LLM 使用時は、どのプロバイダーとモデルを使ったかを運用ログに残せること。

### 可用性
- 主要検索 API は Embedding 無効時でも安定して動作すること。
- LLM / Embedding の外部依存が停止しても、公開カタログ参照 API は継続利用可能であること。
