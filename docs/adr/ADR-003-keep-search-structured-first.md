# ADR-003: 検索は構造化検索を主系統にし、Embedding を補助機能にする

## ステータス
採用

## 背景
MVP の主価値は、MCP サーバーを探し、比較し、導入判断できることにある。
この価値はタグ、認証方式、クライアント対応、ツール名などの構造化情報だけでも十分に成立する。
Embedding を必須にすると、運用コストと外部依存が増える。

## 決定内容
- 検索の主系統はキーワード、タグ、ファセット、関連エントリによる構造化検索とする
- Embedding は後から追加できる補助機能とする
- Embedding が無効でも公開カタログ検索は完全に利用可能でなければならない

## 検討した選択肢
- 初期から Embedding 検索を必須にする
  - 柔軟性は高いが、初期コストが増える
- 構造化検索を主にし、Embedding は任意にする
  - MVP として妥当で、運用負荷も低い

## 影響・トレードオフ
- 自由文の意味検索は後回しになる
- 代わりに、初期段階から安定して検索 API を提供できる
