# ADR-001: Medallion と Ports and Adapters を採用する

## ステータス
採用

## 背景
本システムは、MCP サーバー情報の取り込み、整理、公開、検索を継続的に行う。
生データと公開データを同一モデルで扱うと、変換責務、監査、再整理、将来の差し替えが困難になる。
また、DB、LLM、Embedding の実装は将来的に変更される可能性が高い。

## 決定内容
- データライフサイクルには Bronze / Silver / Gold の Medallion 構造を採用する
- アプリケーション構造には Ports and Adapters を採用する
- ドメインルールは Adapter 実装を知らず、Port 契約だけに依存する

## 検討した選択肢
- 単一 CRUD モデルで管理する
  - 実装は簡単だが、生データと公開データの責務分離が弱い
- レイヤードのみで構成する
  - 一般的だが、DB / LLM / Embedding の差し替え境界が曖昧になりやすい
- Medallion + Ports and Adapters を組み合わせる
  - データ品質段階と依存差し替えの両方を整理しやすい

## 影響・トレードオフ
- モジュール数は増える
- 初期設計コストは上がる
- その代わり、再整理、プロバイダー差し替え、将来の拡張がしやすくなる
