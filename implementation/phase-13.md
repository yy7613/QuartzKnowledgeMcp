# Phase 13: 回帰固定と完成度向上

## 目的
ここまでに積み上げた機能を安定化し、以後の拡張で壊れにくい基準線を作る。

## 最小で動く到達点
主要フローに対する回帰テスト群が揃い、ビルドとテストの安定実行が保証される。

## 実装対象
- 回帰テスト整理
- テストフィクスチャ整理
- カバレッジ方針の明文化
- 実装と docs の差異確認手順
- フェーズ完了報告の運用開始

## 具体的なタスク
1. Bronze -> Silver -> Gold の主要フローを回帰テストで固定する
2. Rule-based / LLM enabled / Embedding disabled の代表ケースを揃える
3. テストデータ生成と Fixture を共通化する
4. カバレッジの最低ラインを決める
5. docs と実装差異の確認手順を完了報告テンプレートへ反映して運用する

## 必須ユニットテスト
- 主要ユースケースの回帰テストが揃っていること
- LLM 無効時と有効時で期待動作が保たれること
- Embedding 無効時でも検索主系統が成立すること

## 確認コマンド
- `dotnet build`
- `dotnet test`

## 突破基準
- 主要機能が回帰テストで守られる
- 今後のリファクタリングに耐える基準線がある
- 実装計画として次の増築へ進める状態になる

## スコープ外
- 本番デプロイ自動化
- 大規模な性能試験

## 備考
このフェーズは機能追加ではなく、完成度と保守性の底上げが主目的である。
