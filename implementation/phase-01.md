# Phase 01: ホスト起動と検証基盤

## 目的
実装を積み上げるための最小ホストと検証基盤を作る。

## 最小で動く到達点
Web API が起動し、ヘルスチェック API が 200 を返し、テストプロジェクトが実行できる。

## 実装対象
- 正式な solution 構成の整理
- Minimal API の最小エントリポイント
- `GET /health` エンドポイント
- テストプロジェクトの基本設定

## 具体的なタスク
1. 使用する solution を 1 つに整理し、以後の作業対象を固定する
2. Minimal API を起動し、`GET /health` を追加する
3. 開発時とテスト時の基本設定を分ける
4. xUnit の土台と共通テスト実行手順を整える

## 必須ユニットテスト
- `HealthStatusService` が正常状態を返すこと
- ヘルスチェック用設定解決が既定値で動くこと

## 確認コマンド
- `dotnet build`
- `dotnet test`

## 突破基準
- API が起動する
- `GET /health` が 200 を返す
- テストプロジェクトが成功する

## スコープ外
- Bronze / Silver / Gold の実装
- DB 永続化
- LLM や Embedding の設定

## 備考
このフェーズでは将来の抽象化より、まず開発と検証が回る足場を優先する。
