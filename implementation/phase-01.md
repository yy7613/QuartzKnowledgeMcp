# Phase 01: スケルトン

## 目的
プロジェクト構成のみを作成し、次フェーズから実装を積み上げられる状態にする。

## 実装対象
- ルート配下のハーネス構成
- `docs` の基本ドキュメント
- `ideas` と `implementation` のテンプレート
- `work` の管理ファイル

## 具体的なタスク
1. ハーネスに必要なフォルダとファイルを作成する
2. `readme.md` と `rules.md` を初期化する
3. `docs` に基本ドキュメントを配置する
4. `work` に差異管理・課題管理ファイルを用意する

## 突破基準
- 記事で定義したフォルダ構成が揃っている
- 次フェーズに必要なテンプレートが存在する
- `src` が存在し、以降の実装を受け入れられる状態になっている

## スコープ外
- 実際のアプリケーションコードの実装
- 技術スタックの確定
- Phase 02 以降の詳細設計

## 備考
ビルドやテストの具体的なコマンドは、技術選定後に `docs/tech-stack.md` と `readme.md` に追記する。
