# Phase 05: カタログ参照と基本検索

## 目的
公開済み Gold エントリを利用者が探せる状態を作る。

## 最小で動く到達点
Gold 一覧、詳細、キーワード検索、タグベースカテゴリ検索、認証 / クライアントの基本フィルタが動く。

## 実装対象
- Gold 一覧 API
- 基本検索 API
- 最小の検索結果 DTO
- 並び替えとページングの基礎
- 認証条件と対応クライアントの基本フィルタ

## 具体的なタスク
1. `GET /api/gold/catalog` を追加する
2. `GET /api/search` を追加する
3. 名前、概要、タグに対する簡易検索を実装する
4. `tags` をカテゴリ絞り込みとして扱う方針を API と実装で統一する
5. `authType` と `client` の基本フィルタを実装する
6. relevance 相当の簡易スコアか更新日時順を定義する
7. ページングの既定値と上限を決める

## 必須ユニットテスト
- キーワード検索が名前と概要に一致すること
- タグ AND 条件が期待通りに絞り込むこと
- 認証条件とクライアント条件で正しく絞り込めること
- 並び順とページングが安定していること

## 確認コマンド
- `dotnet build`
- `dotnet test`

## 突破基準
- Gold 一覧と詳細が利用できる
- 基本検索が利用できる
- タグベースカテゴリ検索と基本フィルタが利用できる
- Embedding なしで探索価値が出る

## スコープ外
- 検索補完
- ファセット
- 関連エントリ

## 備考
ここで検索の主系統を構造化検索として成立させる。MVP のカテゴリはタグで表現する。
