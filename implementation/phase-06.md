# Phase 06: Gold 更新と履歴参照

## 目的
Gold 更新の基本操作を揃え、タグ制約と監査性を満たす。

## 最小で動く到達点
Gold 本体更新、タグの全置換更新、履歴一覧取得が動き、タグは 1 件以上 5 件以下で管理される。

## 実装対象
- Gold 本体更新 API
- タグ更新 API
- 履歴一覧 API
- タグ制約ロジック
- 履歴ページング

## 具体的なタスク
1. `PUT /api/gold/catalog/{entryId}` を追加する
2. `PUT /api/gold/catalog/{entryId}/tags` を追加する
3. `GET /api/gold/catalog/{entryId}/history` を追加する
4. Gold 本体更新時のバリデーションと履歴追記を実装する
5. タグの件数、重複、空文字制約を実装する
6. タグ変更時の履歴追記を実装する
7. Gold 更新時刻の更新を整える

## 必須ユニットテスト
- Gold 本体更新時に対象項目だけが更新されること
- Gold 本体更新時に履歴が追加されること
- タグが 1 件未満または 5 件超で失敗すること
- 重複タグが拒否されること
- タグ更新時に履歴が追加されること
- 履歴一覧がページングできること

## 確認コマンド
- `dotnet build`
- `dotnet test`

## 突破基準
- Gold 本体更新が動く
- タグ更新が動く
- タグ制約が守られる
- 履歴参照が動く

## スコープ外
- 検索補完
- ファセット
- Adapter 抽象化
